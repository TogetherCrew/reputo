# Goal: Review the code like a senior engineer

Steps:

1. Check for:
   - Correctness and edge cases
   - Layering and boundary violations
   - Types and contracts
   - Error handling and logging
   - Performance and scalability
   - Security concerns

Output:

- Issues grouped by severity: blocker / should-fix / nice-to-have.
- Concrete suggestions with file/line pointers where possible.
- "Top 3 things I'd change" shortlist.
